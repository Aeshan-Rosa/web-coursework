Feedback Page

Technical Description

The Feedback page was developed using a combination of HTML5 semantic elements and CSS styling to create an intuitive and user-friendly form interface. The page structure utilizes fieldset elements with distinct legend tags to organize form sections logically (personal information, feedback details, user experience, and attachments). Each section is color-coded with different background colors using rgba values (e.g., rgba(70, 130, 180, 0.1)) to visually separate content areas.

The form implements various input types including text, email, tel, textarea, select, checkbox, radio, file, date, and datetime-local to collect comprehensive user feedback. Form validation is enhanced through HTML5 attributes like required and pattern to ensure data integrity. For user interaction, custom styling was applied to form elements with consistent padding and border-radius properties, along with focus states using box-shadow: 0 0 5px rgba(0, 105, 148, 0.5) for improved accessibility.

A standout feature is the interactive star rating system implemented with CSS, using background-image properties and transitions. Submit and cancel buttons have hover effects using transform: translateY(-2px) to provide subtle visual feedback. The form is made responsive with flexible width settings (90% with max-width: 800px) to ensure usability across various devices.

Accessibility

Accessibility was a core consideration in developing the Feedback page. All form controls have properly associated label elements with explicit relationships to their respective input fields. Required fields are clearly marked using the ::after pseudo-element with appropriate visual indicators. Color contrast ratios meet WCAG AA standards for all text elements against their backgrounds.

Keyboard navigation is fully supported with visible focus states for all interactive elements. Form validation provides clear error messages and focuses on the first field with an error when submission is attempted. ARIA attributes are used where necessary to enhance screen reader compatibility, particularly for the custom star rating system which includes aria-label attributes for each rating option.

The responsive design ensures that form elements maintain appropriate sizing and spacing on all devices, preventing touch target issues on mobile. Text inputs have appropriate font sizes to ensure readability, and form instructions use clear, concise language to guide users through completion.

Team Page

Technical Description

The Team page showcases team members using an interactive card-based layout with flip animation effects. The HTML structure utilizes semantic elements for proper document outline, with nested div elements creating the card interface components. Each team member card contains front and back faces, with the front displaying a profile image and basic information, while the back reveals detailed responsibilities and expertise.

CSS techniques applied include perspective-based 3D animations using transform: rotateY(180deg) for the card flipping effect. The layout is made responsive through Flexbox properties (display: flex, flex-wrap: wrap), allowing cards to reflow naturally across different screen sizes. Visual depth is created with box-shadow: 0 4px 8px rgba(0, 0, 0, 0.1) applied to cards, enhanced on hover with subtle scaling (transform: scale(1.05)).

Interactive elements use smooth transitions (transition: transform 0.3s ease) for all state changes. The team page implements a branded color scheme with hex colors (#3914ab, #4a7c59, #006994) that aligns with the overall site design. Absolute positioning is used for overlay detail cards (position: absolute) to maintain clean layout while providing additional information on interaction.

Accessibility

The Team page addresses accessibility through several key implementations. All team member images include descriptive alt attributes that convey both visual content and context. The interactive card elements are enhanced with tabindex attributes to ensure proper keyboard navigation sequence, and focus-within selectors trigger the same visual effects as hover states for keyboard users.

Semantic HTML5 structure with appropriate heading hierarchy (h1, h3, h4) maintains content organization for screen readers. Color contrast is maintained at WCAG AA standards across all text content against backgrounds. Interactive elements maintain a consistent focus order that follows the visual layout, and focus states are made highly visible with border highlighting.

For users who may have motion sensitivity, the animation effects can be disabled using the CSS prefers-reduced-motion media query, which replaces the flip animation with a simpler opacity transition when selected in user preferences.

Coral Reefs content Page

Technical Description

The Coral Reefs content page was developed as an informative resource combining educational content with engaging visual elements. The HTML structure implements semantic tags including header, main, section, and footer to create a logical document outline. Internal anchor links with descriptive IDs (#biodiversity, #formation, etc.) allow for easy navigation within the extensive content.

CSS techniques include a full-width cover image with overlay text using linear gradient backgrounds (linear-gradient(rgba(0,0,0,0.5), rgba(0,0,0,0.3))) to ensure text readability. The page employs responsive grid layouts with grid-template-columns and auto-fit properties for adaptive content cards. Hover animations utilize transform properties (transform: translateY(-5px)) to create subtle interactive effects when users explore different sections.

Visual hierarchy is established through fact boxes with colored borders and contrasting backgrounds, alongside prominent statistics displays with large numbers and supporting text. The ocean-themed color palette (#006994, #e6f7ff) reinforces the content theme while ensuring visual cohesion with the site's overall design. Navigation is enhanced with a fixed position "back to top" button that uses position: fixed to remain accessible throughout the lengthy content.

Accessibility

The Coral Reefs page implements numerous accessibility features to ensure content availability to all users. Images include comprehensive alt text attributes that describe both the visual content and its contextual significance to the educational material. The text content maintains appropriate heading structure with proper nesting of h1 through h3 elements for clear content hierarchy.

Internal navigation is enhanced with descriptive link text and skip-to-content functionality allowing keyboard users to bypass repetitive navigation elements. Color contrast meets WCAG AA standards for all text content, with special attention to ensuring readability of text overlaid on the cover image through gradient backgrounds. Font Awesome icons used for visual enhancement include appropriate aria-hidden="true" attributes where they are decorative, with additional text descriptions where they convey meaning.

The responsive design ensures content remains accessible at all viewport sizes without horizontal scrolling or overlapping elements. Media queries adjust spacing, font sizes, and grid layouts to maintain readability on smaller screens. Interactive elements maintain visible focus states and sufficient touch target sizes for mobile users.

Challenges and Lessons Learned

During the development process, I encountered several challenges that provided valuable learning opportunities:

Challenges Faced

  • Cross-browser compatibility: Ensuring consistent display across different browsers proved challenging, particularly with the 3D card flip animations on the Team page and custom form elements in the Feedback form. Firefox and Safari handled perspective and transform properties differently than Chrome.
  • Responsive design implementation: Creating a responsive grid layout for the Team cards while maintaining the flip animation effect required careful consideration of viewport sizes and interaction methods.
  • Form validation complexity: Balancing comprehensive validation with user-friendly error handling in the Feedback form required multiple approaches to ensure data integrity without frustrating users.

Solutions Applied

  • Implemented feature detection rather than browser detection, allowing for graceful fallbacks where certain CSS properties weren't fully supported.
  • Adopted a mobile-first approach with progressive enhancement, ensuring core functionality remained intact while adding visual enhancements for larger screens.
  • Combined HTML5 validation attributes with custom validation patterns to provide immediate feedback while maintaining data consistency.

Key Lessons Learned

  • Thorough planning before implementation significantly reduces development time and minimizes the need for major revisions later in the process.
  • Regular user testing throughout development helps identify usability issues early, particularly with interactive elements like the card flip animations.
  • Accessibility considerations should be integrated from the beginning of the design process rather than added as an afterthought.
  • Documentation of code decisions and design choices improves team collaboration and makes future maintenance more efficient.

These experiences have deepened my understanding of front-end development best practices and reinforced the importance of balancing visual design with technical functionality and accessibility.

Compliance

All pages developed comply with Janet regulations governing web page publication through several implemented measures:

  • Acceptable Use Policy (AUP) compliance: All content adheres strictly to Janet's AUP, ensuring educational relevance and appropriate content for an academic environment. The Coral Reefs page in particular focuses on factual, educational content with properly cited sources for all statistical information.
  • Security protocols: The Feedback form implements proper validation techniques to prevent common security vulnerabilities including cross-site scripting (XSS) and cross-site request forgery (CSRF). Input sanitization is applied to all form fields.
  • Data protection: Form submission handling follows data protection principles with clear user consent mechanisms and transparent information about how submitted data will be used.
  • Attribution: All third-party resources used across the pages, including Font Awesome icons, Google Fonts, and images, are properly licensed and attributed within both the code comments and the visible references section.
  • Network efficiency: Image resources are optimized to reduce file sizes and minimize bandwidth usage on the Janet network, with appropriate dimensions and compression levels applied.
  • Accessibility compliance: All pages meet accessibility requirements as specified by Janet regulations, including proper semantic structure, keyboard navigation, and screen reader compatibility.

Regular compliance reviews were conducted throughout development to ensure all pages meet the required standards, and documentation has been maintained to facilitate future updates while preserving compliance.
